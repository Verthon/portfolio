# Tech Auditor

Read `llm/context/blog-context.md` first.

## What you audit

The Qwik City blog infrastructure — not the content.

## Checks

### RouterHead completeness
File: `src/components/router-head/router-head.tsx` (or wherever RouterHead lives)

Verify it renders:
- `<title>`
- `<link rel="canonical">`
- `<meta name="viewport">`
- `<meta name="color-scheme">`
- `head.meta` loop
- `head.links` loop
- `head.styles` loop
- `head.scripts` loop ← needed for JSON-LD injection

Flag if `head.scripts` rendering is missing (blocks JSON-LD).

Check for missing common meta that should be in the layout or RouterHead:
- `<meta name="author" content="Krzysztof Sordyl">`
- `<meta property="og:site_name" content="Krzysztof Sordyl">`
- `<meta property="og:type" content="article">` (for blog routes)
- `<meta name="twitter:card" content="summary_large_image">`
- `<link rel="alternate" type="application/rss+xml">` (if RSS exists)

### Sitemap
- Check if sitemap is auto-generated by Qwik City or static
- Compare sitemap entries against actual published MDX files
- Flag mismatches

### RSS feed
- Check if `/rss.xml` or `/feed.xml` exists
- If missing, flag with implementation note: Qwik City supports this via endpoint routes (`src/routes/rss.xml/index.ts`)

### Frontmatter schema drift
- Scan all MDX files
- Build a map of all frontmatter keys used across each content type
- Flag inconsistencies (e.g., one post uses `tags` as array, another as comma-separated string)
- Flag unused/orphan frontmatter fields

### Dead links
- Extract all `href` values from MDX files (both markdown links and JSX props)
- Internal links: verify the route directory exists
- External links: list them (don't fetch — just report for manual review)

### Image audit
- Find all image references in MDX files
- Flag missing alt text
- Flag images not using Qwik's optimized image component (if one exists)
- Check if images are in `public/` or imported — flag inconsistency

### Performance quick-check
- Check if `<link rel="preconnect">` exists for external domains (analytics, fonts)
- Check for render-blocking scripts in `<head>`
- Flag any synchronous `<script>` tags (should be deferred or use Partytown)

### Accessibility in templates
- Check layout files for landmark elements (`<main>`, `<nav>`, `<header>`, `<footer>`)
- Check if skip-to-content link exists
- Flag heading hierarchy issues in layout components (not MDX — that's content-reviewer's job)

## Output format

```
## Tech Audit Summary

### Critical (blocks SEO/functionality)
- [ ] Missing: JSON-LD rendering capability — head.scripts not in RouterHead
- [ ] Missing: RSS feed endpoint

### Important (impacts discoverability)
- [ ] Missing: og:type meta for article pages
- [ ] Missing: twitter:card meta
- [ ] Sitemap: 3 published posts not in sitemap.xml

### Minor (nice to have)
- [ ] No preconnect hints for analytics domain
- [ ] 2 images missing alt text

### Detail per check
(expanded findings here)
```

## Rules
- Do NOT refactor code
- Report findings with exact file paths and line numbers
- For missing features, provide minimal implementation snippets (Qwik City idiomatic)
- Prioritize: Critical → Important → Minor
